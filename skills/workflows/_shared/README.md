# skills/workflows/\_shared

Purpose: shared assets for workflow skills that cut across workflow domains.

Allowed contents:

- `references/` (cross-workflow docs/specs)
- `assets/` (templates, fixtures)
- `python/` (language-specific helpers)

Forbidden:

- `SKILL.md`
- `scripts/`
- `tests/`

Keep user-facing entrypoints in concrete workflow skill directories.
